# 设备管理模块 API

路径前缀：`/data_management/equipment`。

## POST `/find_equipment_params`

请求字段：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | String | 否 | 设备 ID |
| type | String | 否 | 设备类型 |
| status | String | 否 | 设备状态 |
| area_id | String | 否 | 区域 ID |
| pipe_id | String | 否 | 管线 ID |
| pipe_segment_id | String | 否 | 管段 ID |
| responsible | String | 否 | 负责人 |
| last_check_start | String | 否 | 最近检查开始时间 |
| last_check_end | String | 否 | 最近检查结束时间 |
| page | String | 否 | 页码 |
| page_size | String | 否 | 每页数量 |

返回数组字段：`id`、`type`、`area_name`、`pipe_name`、`pipe_segment_id`、`pipe_segment_name`、`status`、`last_check`、`responsible`。

> 当前后端不返回 `model`、`install_date`。

## POST `/find_equipment_id`

请求体：`{ "id": "1001" }`。返回设备数组。

## POST `/add_equipment`

请求字段：`id`、`area_id`、`pipe_id`、`pipe_segment_id`、`type`、`status`、`responsible`。

## POST `/update_equipment`

请求字段：`id`、`area_id`、`pipe_id`、`pipe_segment_id`、`type`、`status`、`responsible`、`last_overhaul_time`。

## GET `/delete_equipment/{ids}`

`ids` 支持逗号分隔。

## GET `/Indicator_card`

Query：`area_id` 可选，但当前 Controller 调用的是全局统计，`area_id` 暂不参与计算。

返回字段：`total`、`normal`、`fault`、`maintenance`、`offline`。
