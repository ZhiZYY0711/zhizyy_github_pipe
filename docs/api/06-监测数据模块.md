# 监测数据模块 API

路径前缀：`/data_management/monitoring_data`。

## POST `/find_data_params`

请求字段：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| min_pressure / max_pressure | String | 否 | 压力范围 |
| min_flow / max_flow | String | 否 | 流量范围 |
| min_temperature / max_temperature | String | 否 | 温度范围 |
| min_vibration / max_vibration | String | 否 | 振动范围 |
| data_status | String | 否 | 0 安全，1 良好，2 危险，3 高危 |
| sensor_id | String | 否 | 传感器 ID |
| area_id | String | 否 | 区域 ID |
| pipeline_id | String | 否 | 管线 ID |
| pipeline_name | String | 否 | 管线名称 |
| pipe_segment_id | String | 否 | 管段 ID |
| pipe_segment_ids | String[] | 否 | 管段 ID 列表 |
| monitor_start_time | String | 否 | 监测开始时间 |
| monitor_end_time | String | 否 | 监测结束时间 |
| page | Integer | 否 | 页码 |
| page_size | Integer | 否 | 每页数量 |

返回分页记录字段：`id`、`pressure`、`flow`、`temperature`、`vibration`、`data_status`、`status_text`、`sensor_id`、`sensor_name`、`area_name`、`pipeline_name`、`pipe_segment_id`、`pipe_segment_name`、`monitor_time`。

## POST `/find_data_id`

请求体：`{ "id": "1" }`。返回单条监测数据。

## POST `/add_data`

请求字段：`sensor_id`、`flow`、`pressure`、`temperature`、`vibration`、`data_status`、`realtime_picture`。

> 当前新增接口不接收 `monitor_time`。

## POST `/update_data`

请求字段：`id`、`sensor_id`、`flow`、`pressure`、`temperature`、`vibration`。

> 当前更新接口不接收 `data_status`。

## GET `/delete_data/{ids}`

`ids` 支持逗号分隔。

## GET `/indicator_card`

Query：`area_id` 可选。

返回字段：`total_records`、`safe_count`、`good_count`、`danger_count`、`critical_count`、`today`。
