# 区域管理模块 API

## 概述

区域管理模块提供省市区三级联动查询功能。

---

## 1. 获取省份列表

### 接口说明

获取所有省份信息。

### 请求信息

- **URL**: `/area_details/provinces`
- **Method**: `GET`
- **认证**: 需要

### 响应参数

```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "code": "130000",
      "name": "河北省"
    },
    {
      "code": "140000",
      "name": "山西省"
    }
  ]
}
```

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | String | 区域代码 |
| name | String | 区域名称 |

---

## 2. 获取城市列表

### 接口说明

根据省份代码获取城市列表。

### 请求信息

- **URL**: `/area_details/citys/{province_code}`
- **Method**: `GET`
- **认证**: 需要

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| province_code | String | 是 | 省份代码（路径参数） |

### 响应参数

```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "code": "130100",
      "name": "石家庄市"
    },
    {
      "code": "130200",
      "name": "唐山市"
    }
  ]
}
```

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | String | 区域代码 |
| name | String | 区域名称 |

---

## 3. 获取区县列表

### 接口说明

根据城市代码获取区县列表。

### 请求信息

- **URL**: `/area_details/districts/{city_code}`
- **Method**: `GET`
- **认证**: 需要

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| city_code | String | 是 | 城市代码（路径参数） |

### 响应参数

```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "code": "130102",
      "name": "长安区"
    },
    {
      "code": "130104",
      "name": "桥西区"
    }
  ]
}
```

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | String | 区域代码 |
| name | String | 区域名称 |

---

## 区域代码说明

### 编码规则

- **省份代码**: 6位数字，后4位为0000（如：130000 河北省）
- **城市代码**: 6位数字，后2位为00（如：130100 石家庄市）
- **区县代码**: 6位数字（如：130102 长安区）

### 示例

| 代码 | 名称 | 层级 |
|------|------|------|
| 130000 | 河北省 | 省 |
| 130100 | 石家庄市 | 市 |
| 130102 | 长安区 | 区县 |
| 130104 | 桥西区 | 区县 |
