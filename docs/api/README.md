# 管道管理系统 API 接口文档

## 📋 文档概览

本系统是一个管道管理系统，提供管道监测、任务管理、设备管理、虚拟专家咨询等功能。

## 🏗️ 系统架构

- **后端技术栈**: Spring Boot + MyBatis
- **前端技术栈**: Vue 3 + Vite + TypeScript
- **认证方式**: JWT Token
- **实时通信**: SSE (Server-Sent Events)

## 🔐 认证说明

### JWT Token 认证

除了登录接口外，其他接口都需要在请求头中携带 JWT Token：

```
Authorization: Bearer {token}
```

### 获取 Token

通过登录接口获取 Token：

```http
POST /login
Content-Type: application/json

{
  "username": "admin",
  "password": "encrypted_password",
  "code": "123456"
}
```

## 📊 统一响应格式

所有接口统一使用以下响应格式：

### 成功响应

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

### 错误响应

```json
{
  "code": 400,
  "message": "错误信息",
  "data": null
}
```

### 分页响应

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "records": [ ... ],
    "total": 100,
    "pageNum": 1,
    "pageSize": 10
  }
}
```

## 🌐 基础路径

- **开发环境**: `http://localhost:8080`
- **生产环境**: 根据实际部署配置

## 📝 接口命名规范

- 查询接口（非ID）: `POST /模块/find_资源_params`
- 查询接口（ID）: `POST /模块/find_资源_id`
- 新增接口: `POST /模块/add_资源`
- 修改接口: `POST /模块/update_资源`
- 删除接口: `GET /模块/delete_资源/{id}`
- 指标卡接口: `GET /模块/Indicator_card`

## ⚠️ 注意事项

1. 所有时间参数使用时间戳（毫秒）或字符串格式（yyyy-MM-dd HH:mm:ss）
2. 分页参数统一为 `page`（页码，从1开始）和 `page_size`（每页数量）
3. ID 字段以具体后端 DTO 为准，部分接口返回字符串，部分返回数字
4. 区域ID为6位数字编码
