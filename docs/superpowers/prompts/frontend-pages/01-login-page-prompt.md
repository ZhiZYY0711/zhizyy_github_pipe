# 登录页生成提示词

设计一个【油气管网监测系统登录页】，用于【值班人员和管理员安全进入管网监测与调度工作台】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：控制中心前厅（非普通 SaaS 登录页）
- 用户角色：值班人员、管理员
- 核心目标：
  - 目标1：完成身份认证与安全准入
  - 目标2：建立守护基础设施安全运行的使命感
  - 目标3：传达冷硬监控台气质与高端编辑感
- 使用场景：用户首次进入系统或会话过期后重新认证

------

【二、整体布局（Layout System）】

- 顶部结构：无（登录页独立于主系统导航）
- 主体布局：左右双栏（约 58% / 42%）
- 区域划分：
  - 左侧区域（58%）：品牌叙事与图形氛围区
  - 中间核心区：无
  - 右侧区域（42%）：登录卡
- 是否使用特殊布局：无

------

【三、信息架构（Information Architecture）】

模块1：品牌叙事与图形氛围区

- 功能说明：建立视觉识别与使命传达
- 包含信息：系统名称、使命型主文案、少量能力摘要
- 优先级：高（视觉识别层）
- 表达方式：文字要少，重点使用轨道环、信号点阵、管线轮廓、环形指标、状态标识等图案化元素

模块2：登录卡

- 功能说明：身份认证操作区
- 包含信息：账号输入、密码输入、验证码、登录按钮、系统状态、错误提示
- 优先级：高（功能层）

------

【四、核心模块详细设计（Key Module Design）】

模块名称：登录卡

- 展示形式：嵌入式卡片（surface-container-high 层级）
- 每个单元包含：
  - 账号输入框（Recessed 风格，surface-container-lowest 背景，focus 时 secondary 青色发光边框）
  - 密码输入框（同上风格）
  - 验证码区域
  - 主按钮（primary_container #FF6B00 实色，0px 圆角，hover 时 primary 外发光）
- 操作区：
  - 主操作：登录
  - 次操作：验证码刷新
- 状态设计：
  - 认证中：按钮 loading 态
  - 认证失败：内联错误提示（tertiary 危险黄色标识）
  - 系统离线：状态指示灯

------

【五、交互模型（Interaction Model）】

- 主交互方式：表单填写 + 点击提交
- 分层交互：
  - 第一层：视觉氛围感知
  - 第二层：表单操作
  - 第三层：认证结果反馈
- 页面跳转策略：认证成功后跳转至总览仪表盘
- 动态行为：图形氛围区微动效（轨道环旋转、信号点闪烁），认证结果即时反馈

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 登录卡（功能优先）
  2. 品牌氛围区（识别层）
- 设计风格：Tactical Brutalism — 冷硬监控台 + 高端编辑感
- 组件风格：
  - 卡片：0px 圆角，tonal recession 分区，ghost border（outline-variant 20% 透明度）
  - 输入框：Recessed 内嵌风格
  - 按钮：实色 primary_container，hover 外发光
- 状态颜色规范：
  - 正常/在线：secondary 青色
  - 警告：tertiary 危险黄色（脉冲动画 0.8→1.0）
  - 错误：error 红

------

【七、动效设计（Motion & Feedback）】

- Hover 动效：按钮外发光（primary 色阴影），卡片背景偏移至 surface-bright
- 切换动画：Instant（0ms）或 Mechanical（Step-easing），禁止 bouncy/organic
- 加载动画：CRT 扫描线纹理（repeating linear-gradient），轨道环匀速旋转
- 操作反馈：认证失败内联提示，无弹窗

------

【八、扩展能力（Advanced Features，可选）】

- 记住登录状态
- 窄屏响应式堆叠（图形氛围缩减但不消失）

------

【九、设计目标（Design Goals）】

- 1440 宽桌面下首屏完整展示
- 窄屏下平滑堆叠
- 建立使命感而非营销感
- 不能出现通用蓝白后台感，也不能出现"AI 产品官网味"

------

【十、限制与规范（Constraints）】

- 已固定模块，不再推翻左叙事右登录卡结构
- 不再把登录页当作新的视觉试验场
- 颜色遵循已验收设计系统，基底更深、强调色克制
- 不使用圆角（borderRadius: 0px）
- 不使用纯灰色（#808080），使用 slate blue / charcoal 色系
- 所有功能必须基于现有认证接口，不新增后端未提供的认证方式
