# 总览仪表盘生成提示词

设计一个【油气管网监测指挥中枢 Dashboard】，用于【在单屏内完成态势总览、风险识别和处置跟踪】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：Dashboard（指挥中枢）
- 用户角色：区域主干网监测与调度人员
- 核心目标：
  - 目标1：态势总览（一屏掌握全局）
  - 目标2：风险识别（告警与异常即时可见）
  - 目标3：处置跟踪（任务与进度一目了然）
- 使用场景：值班人员进入系统后的首屏工作台，日常巡检与应急响应的起点

------

【二、整体布局（Layout System）】

- 顶部结构：固定主导航（总览 / 监测 / 资产 / 任务 / 应急 / 日志 / 虚拟专家），虚拟专家需有醒目标识但不抢总览主视角
- 主体布局：三栏式（左 / 中 / 右）
- 区域划分：
  - 左侧区域：图形化摘要工作台
  - 中间核心区：地图主画面（唯一主角）
  - 右侧辅助区：风险与处置面板
- 是否使用特殊布局：Canvas 式地图 + 信息簇

------

【三、信息架构（Information Architecture）】

模块1：图形化摘要工作台（左侧）

- 功能说明：当前区域、状态、重点事项的图形化摘要
- 包含信息：环形焦点卡、条形态势卡、雷达式关注卡、轻量切换卡
- 优先级：高
- 表达方式：不要大段文字，用图案化卡片表达

模块2：地图主画面（中部）

- 功能说明：空间关系、热点和点击钻取
- 包含信息：管网地理分布、热点标注、区域/管线钻取入口
- 优先级：最高（唯一主角）

模块3：风险与处置面板（右侧）

- 功能说明：告警、风险、任务与处置进度
- 包含信息：
  - 上方：告警与风险（优先展示）
  - 下方：任务与处置进度
- 优先级：高

------

【四、核心模块详细设计（Key Module Design）】

模块名称：地图主画面

- 展示形式：Canvas 式交互地图
- 每个单元包含：
  - 管线地理走向
  - 区域热点标注（secondary 青色 / tertiary 危险黄色标识）
  - 可点击钻取节点
- 操作区：
  - 主操作：点击区域/管线钻取至监测模块
  - 次操作：缩放、平移
- 状态设计：
  - 正常：secondary 青色标识
  - 异常：tertiary 危险黄色脉冲标识
  - 告警：error 红色标识

------

【五、交互模型（Interaction Model）】

- 主交互方式：点击钻取 / Hover 信息浮层
- 分层交互：
  - 第一层：地图全局浏览
  - 第二层：区域/管线 Hover 摘要
  - 第三层：点击钻取至监测模块详情
- 页面跳转策略：点击地图热点跳转至监测模块，不跳转的情况仅限摘要浮层
- 动态行为：热点脉冲动画，告警实时更新

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 地图主画面（空间关系核心）
  2. 右侧告警与风险（决策优先）
  3. 左侧摘要工作台（辅助感知）
- 设计风格：Tactical Brutalism — 更深色的冷硬监控台
- 组件风格：
  - 卡片：0px 圆角，tonal recession 分区（surface-container-low → surface-container-high）
  - 信息簇：密集数据点 + 冷色负空间
  - 模块边界：ghost border（outline-variant 20%）
- 状态颜色规范：
  - 正常：secondary 青色
  - 警告：tertiary 危险黄色（脉冲 0.8→1.0）
  - 告警/错误：error 红
  - 数据数值：Space Grotesk 等宽字体，subtle text-shadow 模拟磷光

------

【七、动效设计（Motion & Feedback）】

- Hover 动效：卡片背景偏移至 surface-bright，地图节点高亮
- 切换动画：Instant（0ms）或 Mechanical（Step-easing）
- 加载动画：CRT 扫描线纹理全局覆盖
- 操作反馈：告警实时推送，数值变化等宽防跳

------

【八、扩展能力（Advanced Features，可选）】

- 地图点击钻取至监测模块
- 告警推送实时更新
- 热点聚合与摘要推荐

------

【九、设计目标（Design Goals）】

- 首屏必须一次看完，不要底部趋势区，不要纵向滚动条
- 信息密度中高，模块边界明确
- 不能做成普通卡片宫格后台
- 提升态势感知效率

------

【十、限制与规范（Constraints）】

- 已固定模块，不再推翻地图中心结构，不再重做信息架构
- 允许修改仅限：数据接线、细节打磨、小范围视觉微调
- 不使用圆角（borderRadius: 0px）
- 不使用纯灰色（#808080）
- 所有数据必须来自现有后端接口，不新增业务概念
- 不为了页面好看而凭空发明后端没有的数据结构
