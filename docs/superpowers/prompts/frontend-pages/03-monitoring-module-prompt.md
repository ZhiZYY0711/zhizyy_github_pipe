# 监测模块 - 数据监控页面生成提示词

设计一个【油气管网监测数据监控页面】，用于【实时监控管道与区域四维度数据，通过图表可视化发现异常趋势】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：数据监控分析页（B级页面）
- 用户角色：管网监测值班人员、数据分析师
- 核心目标：
  - 目标1：实时监控压力、流量、温度、震动四维度数据
  - 目标2：通过图表可视化快速发现异常趋势
  - 目标3：为深入分析提供筛选和导航入口
- 使用场景：值班人员日常巡检、异常数据初步分析、趋势监控

------

【二、整体布局（Layout System）】

- 顶部结构：固定主导航（总览 / 监测 / 资产 / 任务 / 应急 / 日志 / 虚拟专家）
- 主体布局：左右双栏（约 30% / 70%）
- 区域划分：
  - 左侧区域（30%）：筛选参数控制面板
  - 右侧区域（70%）：图表数据可视化区
- 是否使用特殊布局：单屏布局，无页面纵向滚动

------

【三、信息架构（Information Architecture）】

模块1：筛选参数控制面板（左侧）

- 功能说明：控制数据监控的范围和维度
- 包含信息：区域选择、管道选择、时间范围、数据维度、刷新设置
- 优先级：高（控制层）
- 表达方式：紧凑表单控件，使用设计系统 tokens

模块2：关键指标卡片区（右侧上部）

- 功能说明：展示核心监控指标摘要
- 包含信息：4个KPI卡片（管道总数、在线监测点、高风险告警、活跃任务）
- 优先级：高（摘要层）

模块3：多维数据图表区（右侧中部）

- 功能说明：可视化展示四维度数据趋势
- 包含信息：多维度折线图（支持压力、流量、温度、震动切换）
- 优先级：最高（可视化核心）

模块4：数据状态分布区（右侧下部）

- 功能说明：展示数据状态分类统计
- 包含信息：饼图或环形图（安全、良好、危险、高危数据分布）
- 优先级：中（辅助分析）

模块5：导航操作区（右侧右下角）

- 功能说明：提供深入分析入口
- 包含信息："查看详情"按钮（跳转到监测数据页面）
- 优先级：中（导航层）

------

【四、核心模块详细设计（Key Module Design）】

模块名称：多维数据图表区

- 展示形式：交互式折线图（支持多系列切换）
- 每个图表包含：
  - X轴：时间序列
  - Y轴：数据值（压力MPa、流量m³/h、温度℃、震动mm/s）
  - 系列：支持同时显示1-4个数据维度
  - 网格线：使用 `--color-line` 低透明度
- 操作区：
  - 主操作：维度切换按钮组
  - 次操作：时间范围缩放、图表重置
- 状态设计：
  - 正常范围：`--color-accent-cyan` 青色线条
  - 警告范围：`--color-warning` 黄色线条
  - 危险范围：`--color-accent-orange` 橙色线条

模块名称：筛选参数控制面板

- 展示形式：垂直堆叠的表单控件组
- 每个控件包含：
  - 区域选择器：多选下拉，支持按省市筛选
  - 管道选择器：多选下拉，支持按管道名称筛选
  - 时间范围选择器：快捷选项（近1小时、近6小时、近24小时、自定义）
  - 数据维度选择：复选框组（压力、流量、温度、震动）
  - 刷新频率：单选组（实时、30秒、1分钟、5分钟、手动）
- 操作区：
  - 主操作：应用筛选按钮
  - 次操作：重置筛选按钮
- 状态设计：
  - 活动筛选：使用 `--color-accent-cyan-soft` 背景高亮
  - 无效状态：使用 `--color-text-dim` 文字

------

【五、交互模型（Interaction Model）】

- 主交互方式：表单筛选 + 图表交互
- 分层交互：
  - 第一层：筛选参数设置
  - 第二层：图表数据浏览（hover查看具体值）
  - 第三层：维度切换和图表操作
- 页面跳转策略：点击"查看详情"按钮跳转到监测数据页面
- 动态行为：实时数据更新、图表动画、筛选应用loading状态

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 多维数据图表（可视化核心）
  2. 关键指标卡片（决策摘要）
  3. 筛选控制面板（操作控制）
  4. 数据状态分布（辅助分析）
- 设计风格：Tactical Brutalism — 冷硬监控台
- 组件风格：
  - 面板：0px圆角，`--color-panel` 背景，`--color-line` 边框
  - 卡片：`--color-panel-2` 背景，紧凑内边距
  - 图表：深色背景，高对比度数据线
- 状态颜色规范：
  - 正常/安全：`--color-accent-cyan` 青色
  - 警告/良好：`--color-warning` 黄色
  - 危险：`--color-accent-orange` 橙色
  - 高危：`--color-accent-orange` 橙色（更高饱和度）
  - 数据数值：使用 `--font-mono` 等宽字体

------

【七、动效设计（Motion & Feedback）】

- Hover动效：图表数据点高亮，筛选控件轻微背景色变化
- 切换动画：图表数据更新使用淡入淡出，维度切换使用横向滑动
- 加载动画：数据加载时显示CRT扫描线纹理，图表区域骨架屏
- 操作反馈：筛选应用成功提示，数据更新toast通知

------

【八、扩展能力（Advanced Features，可选）】

- 图表对比模式（支持多区域/多管道对比）
- 异常数据自动标注
- 趋势预测线显示
- 数据导出（图表截图、数据CSV）

------

【九、设计目标（Design Goals）】

- 单屏内完成所有监控功能，无需纵向滚动
- 提升异常数据发现效率
- 降低多维度数据认知成本
- 保持与Dashboard首页的视觉连续性

------

【十、限制与规范（Constraints）】

- 必须使用已验收的设计系统 tokens
- 颜色遵循 `--color-*` 变量，不使用硬编码色值
- 字体遵循 `--font-*` 体系，数据使用等宽字体
- 不使用圆角（borderRadius: 0px）
- 所有数据必须来自现有后端接口：
  - 区域数据：`POST /data_visualization/data_monitoring/area_data`
  - 管道数据：`POST /data_visualization/data_monitoring/pipe_data`
  - 关键指标：`GET /data_visualization/data_monitoring/key_indicators`
  - 不新增后端未提供的业务概念
- 页面必须保持冷硬监控台气质，不做营销式视觉设计

==================================================

【API集成说明】

1. 页面初始化时调用：
   - `GET /data_visualization/data_monitoring/key_indicators` 获取关键指标
   - 默认显示华北区域数据

2. 筛选变化时调用：
   - 区域筛选：`POST /data_visualization/data_monitoring/area_data`
   - 管道筛选：`POST /data_visualization/data_monitoring/pipe_data`

3. 实时更新：
   - 根据刷新频率定时调用相应接口
   - 支持WebSocket推送（如后端支持）

4. 跳转到监测数据页面时传递当前筛选参数
