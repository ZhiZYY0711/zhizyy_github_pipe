# 监测模块生成提示词

设计一个【监测模块页面组】，用于【替代旧系统中区域详情页、管线详情页和监测数据页，实现图表先行、明细后置的监控分析流程】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：空间式态势面 + 高密度数据页（双页组合）
- 用户角色：监测分析人员
- 核心目标：
  - 目标1：先从空间和图表概览进入（图表先行）
  - 目标2：再逐步进入对象级细节（明细后置）
  - 目标3：不退化成单纯二维表视图，也不无节制叠加图表
- 使用场景：值班人员从总览钻取进入，或主动进入监测模块进行区域/管线分析

------

【二、整体布局（Layout System）】

本模块包含两个页面：

**页面A：数据监控页**

- 顶部结构：固定主导航 + 模块内子导航
- 主体布局：Fullpage Section Scrolling + Scroll Snap
- 区域划分：
  - 每一屏为一个独立主题段落
  - 按屏切换，全屏分屏滚动
- 是否使用特殊布局：Fullpage Section Scrolling + Scroll Snap

**页面B：监测数据页**

- 顶部结构：固定主导航 + 筛选工作区
- 主体布局：上下分层（卡片摘要区 + 高密度表格区）
- 区域划分：
  - 上层：高危数据卡、异常摘要块、高风险推荐区
  - 下层：高密度二维数据表
- 是否使用特殊布局：卡片流 + Data Grid

------

【三、信息架构（Information Architecture）】

**页面A：数据监控页**

模块1：区域态势屏

- 功能说明：区域整体运行态势概览
- 包含信息：区域关键指标、空间分布图、状态摘要
- 优先级：最高（入口屏）

模块2：区域异常聚合屏

- 功能说明：区域内异常链路与处置关联聚合
- 包含信息：异常分布、告警聚合、处置状态
- 优先级：高

模块3：管线焦点屏

- 功能说明：重点管段聚焦展示
- 包含信息：关键管线状态、压力/流量趋势
- 优先级：高

模块4：管线细节屏

- 功能说明：对象级细节下钻
- 包含信息：单条管线详细数据、历史对比、监测点详情
- 优先级：中

**页面B：监测数据页**

模块1：高危数据卡片区

- 功能说明：前置筛选，突出危险数据和高危数据
- 包含信息：危险数据卡、异常摘要块、高风险推荐区
- 优先级：最高

模块2：全量数据表

- 功能说明：承接全部监测数据
- 包含信息：高密度二维数据表
- 优先级：高

------

【四、核心模块详细设计（Key Module Design）】

模块名称：区域态势屏（数据监控页第一屏）

- 展示形式：空间式态势面
- 每个单元包含：
  - 区域关键指标（Space Grotesk 等宽数值 + subtle text-shadow）
  - 空间分布图（地图/拓扑）
  - 状态摘要卡（tonal recession 分区）
- 操作区：
  - 主操作：Scroll Snap 下滑至下一屏
  - 次操作：点击指标钻取至监测数据页
- 状态设计：
  - 正常：secondary 青色
  - 异常：tertiary 危险黄色脉冲
  - 告警：error 红

模块名称：高危数据卡片区（监测数据页上层）

- 展示形式：卡片突出
- 每个单元包含：
  - 危险等级标识（tertiary 危险黄色 / error 红）
  - 关键数值（Space Grotesk 等宽）
  - 简要描述
  - 钻取链接
- 操作区：
  - 主操作：点击卡片查看详情
  - 次操作：筛选过滤
- 状态设计：
  - 危险：error 红 + 脉冲动画
  - 高危：tertiary 危险黄色 + 脉冲动画

------

【五、交互模型（Interaction Model）】

- 主交互方式：
  - 数据监控页：Scroll Snap 按屏滚动
  - 监测数据页：卡片点击 + 表格筛选
- 分层交互：
  - 第一层：数据监控页全局概览（图表先行）
  - 第二层：逐屏下钻至对象级
  - 第三层：跳转至监测数据页查看明细
- 页面跳转策略：
  - 数据监控页 → 监测数据页（图表先行，明细后置）
  - 不允许反向跳转覆盖
- 动态行为：Scroll Snap 吸附，数值实时更新

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 区域态势屏（入口屏，叙事起点）
  2. 高危数据卡片区（决策优先）
  3. 全量数据表（明细承接）
- 设计风格：Tactical Brutalism — 数据监控页偏空间式与叙事式，监测数据页偏卡片摘要 + 高密度表格
- 组件风格：
  - 卡片：0px 圆角，tonal recession 分区
  - 数据表：Data Grid（label-sm 密集矩阵，outline-variant 10% 纵向分隔线每4列）
  - 数值：Space Grotesk 等宽，text-shadow 模拟磷光
- 状态颜色规范：
  - 正常：secondary 青色
  - 警告/高危：tertiary 危险黄色（脉冲 0.8→1.0）
  - 危险/告警：error 红

------

【七、动效设计（Motion & Feedback）】

- Hover 动效：卡片背景偏移至 surface-bright，表格行高亮
- 切换动画：Scroll Snap 吸附（Mechanical Step-easing）
- 加载动画：CRT 扫描线纹理
- 操作反馈：数值变化等宽防跳，告警脉冲

------

【八、扩展能力（Advanced Features，可选）】

- 高危数据推荐区（展示层聚合，非新增业务）
- 异常摘要索引
- 区域/管线快速切换

------

【九、设计目标（Design Goals）】

- 图表先行，明细后置
- 数据监控页：带叙事节奏的监控分析页，不是普通详情页
- 监测数据页：先概览再明细，不是纯二维表
- 提升从全局到细节的下钻效率

------

【十、限制与规范（Constraints）】

- 业务来源：原项目数据可视化中的数据监控模块 + 原区域详情页 + 原管线详情页 + 监测数据模块
- 所有功能必须建立在现有监测接口之上，不新增业务概念，只重组信息视图
- 不把所有内容都塞在一个二维表里
- 也不完全取消二维表
- 不为了叙事感而杜撰接口返回内容
- 不使用圆角（borderRadius: 0px）
- 不使用纯灰色（#808080）
