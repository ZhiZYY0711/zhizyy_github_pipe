# 监测模块 - 监测数据页面生成提示词

设计一个【油气管网监测数据详情页面】，用于【深入分析监测数据，重点关注危险和高危数据，提供完整数据浏览和筛选能力】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：数据详情分析页（C级页面）
- 用户角色：管网监测值班人员、数据分析师、调度指挥人员
- 核心目标：
  - 目标1：突出展示危险和高危监测数据，确保异常不被遗漏
  - 目标2：提供高密度数据浏览和高级筛选能力
  - 目标3：支持数据导出和深入分析操作
- 使用场景：从数据监控页面跳转而来，进行异常数据深入分析、数据审计、报告生成

------

【二、整体布局（Layout System）】

- 顶部结构：固定主导航（总览 / 监测 / 资产 / 任务 / 应急 / 日志 / 虚拟专家） + 面包屑导航
- 主体布局：上下分层结构
- 区域划分：
  - 上部区域（约30%高度）：危险数据突出展示区
  - 下部区域（约70%高度）：高密度数据表格区
- 是否使用特殊布局：支持分页模式和长页面滚动模式切换

------

【三、信息架构（Information Architecture）】

模块1：危险数据突出展示区（上部）

- 功能说明：优先展示需要立即关注的异常数据
- 包含信息：危险数据卡片（数据状态=2）、高危数据卡片（数据状态=3）
- 优先级：最高（风险预警层）
- 表达方式：醒目卡片式展示，使用语义色强调

模块2：数据筛选控制栏（上部下方）

- 功能说明：提供高级数据筛选能力
- 包含信息：状态筛选、时间范围、数值范围、管道筛选、搜索框
- 优先级：高（控制层）
- 表达方式：紧凑水平筛选栏，支持展开高级筛选

模块3：数据表格展示区（下部）

- 功能说明：展示完整监测数据，支持多种浏览模式
- 包含信息：高密度二维表格，包含所有监测数据字段
- 优先级：高（数据核心层）
- 表达方式：可切换分页模式/滚动模式，支持列自定义

模块4：表格操作栏（表格上方）

- 功能说明：提供表格操作和视图控制
- 包含信息：视图模式切换、列显示控制、数据导出、刷新按钮
- 优先级：中（操作层）

------

【四、核心模块详细设计（Key Module Design）】

模块名称：危险数据突出展示区

- 展示形式：横向卡片流（每行2-4张卡片）
- 每张卡片包含：
  - 标题：传感器名称 + 管道名称
  - 核心指标：异常数值（如：压力 1.8MPa）
  - 状态标签：危险/高危 + 持续时间
  - 简要描述：异常现象描述
  - 建议操作：立即查看、标记处理、忽略
- 操作区：
  - 主操作：点击卡片查看详细数据
  - 次操作：标记为已处理、添加到关注列表
- 状态设计：
  - 危险数据：`--color-warning` 黄色主题，轻微脉冲动画
  - 高危数据：`--color-accent-orange` 橙色主题，明显脉冲动画
  - 已处理：`--color-text-dim` 灰色主题，降低透明度

模块名称：高密度数据表格

- 展示形式：可配置二维表格
- 每行包含：
  - 传感器ID：`--font-mono` 等宽字体
  - 传感器名称
  - 管道名称
  - 压力(MPa)：数值 + 状态色标
  - 流量(m³/h)：数值 + 状态色标
  - 温度(℃)：数值 + 状态色标
  - 震动(mm/s)：数值 + 状态色标
  - 数据状态：标签形式（安全、良好、危险、高危）
  - 监测时间：YYYY-MM-DD HH:mm:ss
  - 操作列：查看详情、数据对比、标记
- 操作区：
  - 行操作：点击行选中，双击查看详情
  - 列操作：点击列头排序，拖拽调整列宽
  - 批量操作：多选后批量导出、批量标记
- 状态设计：
  - 选中行：`--color-accent-cyan-soft` 背景高亮
  - 悬停行：`--color-panel-3` 背景
  - 状态色标：根据数值范围显示不同颜色

------

【五、交互模型（Interaction Model）】

- 主交互方式：表格浏览 + 筛选控制
- 分层交互：
  - 第一层：危险数据卡片快速处理
  - 第二层：筛选条件设置和数据范围缩小
  - 第三层：表格数据详细浏览和操作
- 页面跳转策略：从数据监控页面跳转而来，支持返回数据监控页面
- 动态行为：
  - 分页模式：传统分页，显示页码和每页条数
  - 滚动模式：无限滚动，自动加载更多数据
  - 实时更新：新数据插入提示，危险数据实时高亮

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 危险数据卡片（风险预警）
  2. 数据筛选控制（操作控制）
  3. 数据表格（数据核心）
  4. 表格操作栏（辅助操作）
- 设计风格：Tactical Brutalism — 冷硬监控台的高密度变体
- 组件风格：
  - 危险卡片：`--color-panel-2` 背景，语义色边框，紧凑内边距
  - 筛选栏：`--color-panel` 背景，内联表单控件
  - 数据表格：`--color-bg-elevated` 背景，细边框分隔，紧凑行高
- 状态颜色规范：
  - 安全数据：`--color-success` 绿色
  - 良好数据：`--color-accent-cyan` 青色
  - 危险数据：`--color-warning` 黄色
  - 高危数据：`--color-accent-orange` 橙色
  - 表格文字：`--color-text` 主文本，`--font-sans` 字体
  - 数据数值：`--font-mono` 等宽字体，`--color-text` 颜色

------

【七、动效设计（Motion & Feedback）】

- Hover动效：表格行背景色变化，卡片轻微上浮
- 切换动画：筛选条件应用时表格数据淡入更新
- 加载动画：数据加载时显示进度条，危险卡片脉冲动画
- 操作反馈：数据标记成功toast，导出操作进度提示

------

【八、扩展能力（Advanced Features，可选）】

- 数据对比模式（支持多行数据对比）
- 自定义视图保存和加载
- 数据趋势分析（选中数据生成趋势图）
- 批量数据导出（支持Excel、CSV、PDF）
- 数据订阅提醒（特定条件数据自动推送）

------

【九、设计目标（Design Goals）】

- 确保危险和高危数据不被遗漏，优先展示
- 提供高效的数据筛选和浏览体验
- 支持大规模数据处理（万级以上数据量）
- 保持与数据监控页面的操作连续性
- 在C级页面中保持设计系统一致性

------

【十、限制与规范（Constraints）】

- 必须使用已验收的设计系统 tokens
- 颜色严格遵循语义色规范，不自定义颜色
- 字体系统保持一致，表格数据使用等宽字体
- 不使用圆角（borderRadius: 0px）
- 所有数据必须来自现有后端接口：
  - 监测数据查询：`POST /data_management/monitoring_data/find_data_params`
  - 监测数据指标：`GET /data_management/monitoring_data/indicator_card`
  - 支持所有筛选参数（压力范围、流量范围、温度范围、震动范围、数据状态等）
  - 不新增后端未提供的筛选条件
- 页面作为C级工作流页面，风格表达收敛，优先保证操作效率
- 表格设计遵循高密度原则，但保证可读性

==================================================

【API集成说明】

1. 页面初始化时调用：
   - `GET /data_management/monitoring_data/indicator_card` 获取数据统计指标
   - `POST /data_management/monitoring_data/find_data_params` 获取初始数据（默认显示危险和高危数据）

2. 筛选条件变化时调用：
   - 实时调用 `POST /data_management/monitoring_data/find_data_params` 获取筛选后数据
   - 支持防抖处理，避免频繁请求

3. 视图模式切换：
   - 分页模式：传递 page 和 page_size 参数
   - 滚动模式：使用游标或页码递增加载

4. 数据操作：
   - 数据标记：调用更新接口（如支持）
   - 数据导出：前端生成或请求后端生成

5. 从数据监控页面跳转时：
   - 接收筛选参数（区域、管道、时间范围等）
   - 应用这些参数作为初始筛选条件
   - 在面包屑中显示返回链接

【数据状态映射】
- 数据状态值：0=安全，1=良好，2=危险，3=高危
- 前端显示：对应中文标签和语义色
- 默认筛选：优先显示状态=2和状态=3的数据
