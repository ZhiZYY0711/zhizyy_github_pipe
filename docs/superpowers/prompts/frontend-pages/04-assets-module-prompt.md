# 资产模块生成提示词

设计一个【资产模块工作台】，用于【统一查看设备信息与检修员信息，实现首页有限展示、全量表格承接、抽屉详情查看的高效管理流程】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：卡片式工作台 + 高密度二维表（双页组合）
- 用户角色：设备与人员管理人员
- 核心目标：
  - 目标1：首页用有限空间做重点呈现（受控高度）
  - 目标2：全量数据在专门页用二维表承接
  - 目标3：详情统一右侧抽屉，不新增独立详情页
- 使用场景：管理人员日常查看设备状态、检修员资源分布，快速定位与筛选

------

【二、整体布局（Layout System）】

本模块包含两个页面层级：

**页面A：资产工作台首页**

- 顶部结构：固定主导航 + 一键跳转区（设备信息 / 检修员信息快速定位）
- 主体布局：单栏上下滚动（普通滚动，非 Fullpage）
- 区域划分：
  - 上方：设备信息区
  - 下方：检修员信息区
  - 一键跳转区：两个分区之间快速定位
- 是否使用特殊布局：卡片流

**页面B：全量数据页**

- 顶部结构：固定主导航 + 筛选工作区
- 主体布局：高密度二维表
- 区域划分：全屏表格
- 是否使用特殊布局：Data Grid

------

【三、信息架构（Information Architecture）】

模块1：设备信息区

- 功能说明：设备状态摘要与重点设备关注
- 包含信息：设备卡片流、状态摘要、重点设备关注区
- 优先级：高

模块2：检修员信息区

- 功能说明：检修员资源分布与状态
- 包含信息：检修员卡片流、资源分布区、状态摘要
- 优先级：高

模块3：全量数据表（全量数据页）

- 功能说明：承接超出首页承载的全部数据
- 包含信息：高密度二维表（设备或检修员全量）
- 优先级：中

------

【四、核心模块详细设计（Key Module Design）】

模块名称：设备信息区（首页）

- 展示形式：卡片流 + 状态摘要
- 每个单元包含：
  - 设备名称/编号（Space Grotesk 等宽）
  - 运行状态标识（secondary 青色 / tertiary 危险黄色 / error 红）
  - 关键参数摘要
- 操作区：
  - 主操作：点击卡片 → 右侧抽屉打开详情
  - 次操作："查看全部" → 跳转全量数据页
- 状态设计：
  - 正常运行：secondary 青色
  - 需关注：tertiary 危险黄色
  - 故障：error 红

模块名称：检修员信息区（首页）

- 展示形式：卡片流 + 资源分布
- 每个单元包含：
  - 检修员姓名/工号
  - 在岗状态
  - 负责区域/设备
- 操作区：
  - 主操作：点击卡片 → 右侧抽屉打开详情
  - 次操作："查看全部" → 跳转全量数据页

------

【五、交互模型（Interaction Model）】

- 主交互方式：卡片点击 → 右侧抽屉
- 分层交互：
  - 第一层：首页卡片浏览（有限展示）
  - 第二层：右侧抽屉查看详情
  - 第三层："查看全部"跳转全量数据页
- 页面跳转策略：
  - 首页 → 全量数据页（"查看全部"按钮）
  - 详情不跳转，统一右侧抽屉
- 动态行为：抽屉滑入/滑出，卡片 Hover 高亮

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 设备信息区（核心资产）
  2. 检修员信息区（人力资源）
  3. 全量数据表（明细承接）
- 设计风格：Tactical Brutalism — 冷硬监控台语言，交互以高效管理为主，少做装饰
- 组件风格：
  - 卡片：0px 圆角，tonal recession 分区
  - 抽屉：surface-bright + backdrop-filter: blur(12px)
  - 全量数据表：Data Grid（label-sm 密集矩阵，outline-variant 10% 纵向分隔线每4列）
- 状态颜色规范：
  - 正常：secondary 青色
  - 需关注：tertiary 危险黄色
  - 故障/错误：error 红

------

【七、动效设计（Motion & Feedback）】

- Hover 动效：卡片背景偏移至 surface-bright
- 切换动画：抽屉滑入（Mechanical Step-easing）
- 加载动画：CRT 扫描线纹理
- 操作反馈：抽屉打开/关闭即时响应

------

【八、扩展能力（Advanced Features，可选）】

- 一键跳转区快速定位（设备/检修员分区切换）
- 重点设备关注区（展示层聚合）
- 检修资源分布可视化

------

【九、设计目标（Design Goals）】

- 首页受控高度，单容器最多一页半
- 高效管理为主，多做筛选与快速定位
- 不是把设备和检修员做成两个独立后台页
- 提升资产检索与状态感知效率

------

【十、限制与规范（Constraints）】

- 业务来源：原项目"数据管理"中的设备信息模块 + 检修员信息模块
- 首页不能把所有卡片无节制塞满
- 单个大容器高度最多控制在一页半
- 超出时只允许：分页 或 "查看全部"按钮跳转到新页面
- 全量数据页必须使用高密度二维表
- 详情统一右侧抽屉，不新增独立详情页
- 所有功能必须基于现有设备管理和检修员管理接口
- 不使用圆角（borderRadius: 0px）
- 不使用纯灰色（#808080）
- 不为了交互完整而凭空增加后端没有的操作流
