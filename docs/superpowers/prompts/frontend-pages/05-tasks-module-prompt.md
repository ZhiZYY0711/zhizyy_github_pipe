# 任务模块生成提示词

设计一个【任务模块工作台】，用于【管理巡检、复核、派发和闭环任务，实现首页受控展示、全量表格承接、抽屉详情查看的高效任务调度流程】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：卡片式工作台 + 高密度二维表（双页组合）
- 用户角色：任务调度和执行跟踪人员
- 核心目标：
  - 目标1：首页受控高度，突出任务切片（待处理/处理中/逾期/已完成）
  - 目标2：全量任务在专门页用二维表承接
  - 目标3：详情统一右侧抽屉，不新增独立详情页
- 使用场景：任务调度人员日常分派与跟踪，执行人员查看待办与完成情况

------

【二、整体布局（Layout System）】

本模块包含两个页面层级：

**页面A：任务工作台首页**

- 顶部结构：固定主导航 + 任务切片切换（待处理 / 处理中 / 逾期 / 已完成）
- 主体布局：单栏上下滚动
- 区域划分：
  - 任务切片区：卡片式任务摘要
  - 时间顺序任务流 / 优先级看板
- 是否使用特殊布局：卡片流 / 看板

**页面B：全量任务列表页**

- 顶部结构：固定主导航 + 筛选工作区
- 主体布局：高密度二维表
- 区域划分：全屏表格
- 是否使用特殊布局：Data Grid

------

【三、信息架构（Information Architecture）】

模块1：任务切片区（首页）

- 功能说明：按状态分类的任务摘要展示
- 包含信息：待处理、处理中、逾期、已完成任务卡片
- 优先级：最高

模块2：时间顺序任务流 / 优先级看板（首页）

- 功能说明：任务的时间线或优先级视角
- 包含信息：任务时间节点、优先级排序
- 优先级：高

模块3：全量任务表（全量列表页）

- 功能说明：承接超出首页承载的全部任务
- 包含信息：高密度二维表（全量任务）
- 优先级：中

------

【四、核心模块详细设计（Key Module Design）】

模块名称：任务切片区（首页）

- 展示形式：卡片式任务摘要
- 每个单元包含：
  - 任务名称/编号（Space Grotesk 等宽）
  - 状态标识（待处理 / 处理中 / 逾期 / 已完成）
  - 优先级标识
  - 截止时间
  - 执行人摘要
- 操作区：
  - 主操作：点击卡片 → 右侧抽屉打开详情
  - 次操作："查看全部" → 跳转全量列表页
- 状态设计：
  - 待处理：secondary 青色
  - 处理中：primary 橙色
  - 逾期：tertiary 危险黄色（脉冲 0.8→1.0）
  - 已完成：surface 静默态

------

【五、交互模型（Interaction Model）】

- 主交互方式：卡片点击 → 右侧抽屉
- 分层交互：
  - 第一层：首页任务切片浏览（受控展示）
  - 第二层：右侧抽屉查看任务详情
  - 第三层："查看全部"跳转全量列表页
- 页面跳转策略：
  - 首页 → 全量列表页（"查看全部"按钮）
  - 详情不跳转，统一右侧抽屉
- 动态行为：抽屉滑入/滑出，切片切换，逾期任务脉冲提醒

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 逾期任务（最需关注）
  2. 待处理任务（待决策）
  3. 处理中任务（跟踪中）
  4. 已完成任务（归档）
- 设计风格：Tactical Brutalism — 冷硬监控台，突出快速决策与高效操作
- 组件风格：
  - 卡片：0px 圆角，tonal recession 分区
  - 抽屉：surface-bright + backdrop-filter: blur(12px)
  - 全量列表：Data Grid（label-sm 密集矩阵，outline-variant 10% 纵向分隔线每4列）
- 状态颜色规范：
  - 待处理：secondary 青色
  - 处理中：primary 橙色
  - 逾期：tertiary 危险黄色（脉冲）
  - 已完成：surface 静默

------

【七、动效设计（Motion & Feedback）】

- Hover 动效：卡片背景偏移至 surface-bright
- 切换动画：抽屉滑入（Mechanical Step-easing），切片切换 Instant
- 加载动画：CRT 扫描线纹理
- 操作反馈：逾期任务脉冲提醒，任务状态变更即时反馈

------

【八、扩展能力（Advanced Features，可选）】

- 任务优先级看板视图
- 时间顺序任务流视图
- 任务切片快速筛选

------

【九、设计目标（Design Goals）】

- 首页受控高度，不能无限下拉堆任务
- 突出快速决策与高效操作
- 不做营销式数据大屏
- 不需要重新发明全新流程，只需把原任务管理模块在新设计系统中重新组织表达

------

【十、限制与规范（Constraints）】

- 业务来源：原项目"数据管理"中的任务管理模块
- 继承资产模块的组织策略：首页受控展示 + 全量表格 + 抽屉详情
- 超过首页承载的全量任务必须进入单独列表页
- 列表页使用高密度二维表
- 任务详情通过右侧抽屉展开
- 不新增后端未提供的任务流能力
- 不使用圆角（borderRadius: 0px）
- 不使用纯灰色（#808080）
- 不为了交互完整而凭空增加后端没有的操作流
