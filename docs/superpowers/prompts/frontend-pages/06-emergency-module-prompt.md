# 应急模块生成提示词

设计一个【应急模块】，用于【展示模拟与演练过程，以时间流为核心组织方式呈现演练阶段、时间节点、参与对象和当前状态】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：时间流工作面
- 用户角色：演练与应急指挥人员
- 核心目标：
  - 目标1：以时间流为核心组织演练信息
  - 目标2：多样化展示演练阶段、时间节点、参与对象、当前状态、历史记录
  - 目标3：以时间为核心的筛选机制
- 使用场景：指挥人员查看演练进展、回溯历史演练、按时间筛选演练事件

------

【二、整体布局（Layout System）】

- 顶部结构：固定主导航 + 时间筛选器
- 主体布局：时间流 + 侧边摘要
- 区域划分：
  - 左侧/主区域：纵向时间流（时间片、节点卡片、阶段状态）
  - 右侧辅助区：演练摘要、关键节点放大块
- 是否使用特殊布局：Timeline（纵向时间流）

------

【三、信息架构（Information Architecture）】

模块1：时间流主视图

- 功能说明：以时间为核心组织演练过程
- 包含信息：演练阶段、时间节点、参与对象、当前状态、关键处置节点
- 优先级：最高

模块2：侧边摘要区

- 功能说明：演练概览与关键节点放大
- 包含信息：场景卡片、关键节点放大块、参与区域摘要
- 优先级：高

模块3：筛选器

- 功能说明：以时间为核心的筛选机制
- 包含信息：日期范围、演练类型、参与区域等维度
- 优先级：高

------

【四、核心模块详细设计（Key Module Design）】

模块名称：时间流主视图

- 展示形式：纵向时间流（Timeline）
- 每个单元包含：
  - 时间节点（Space Grotesk 等宽）
  - 阶段状态标识（secondary 青色 / primary 橙色 / tertiary 危险黄色）
  - 参与对象摘要
  - 关键处置节点描述
  - 当前状态标签
- 操作区：
  - 主操作：点击节点卡片展开详情
  - 次操作：时间筛选
- 状态设计：
  - 未开始：surface 静默
  - 进行中：primary 橙色
  - 已完成：secondary 青色
  - 异常/逾期：tertiary 危险黄色脉冲

------

【五、交互模型（Interaction Model）】

- 主交互方式：时间流滚动 + 节点点击展开
- 分层交互：
  - 第一层：时间流全局浏览
  - 第二层：节点卡片展开详情
  - 第三层：关键节点放大查看
- 页面跳转策略：不跳转，全部在当前页面展开
- 动态行为：时间流滚动，节点展开/收起，筛选即时过滤

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 当前进行中的演练节点（决策优先）
  2. 异常/逾期节点
  3. 历史已完成节点
- 设计风格：Tactical Brutalism — 重点不是地图总览，而是过程与阶段感
- 组件风格：
  - 时间节点卡片：0px 圆角，tonal recession 分区
  - 时间轴线：outline-variant 20% ghost border
  - 阶段标识：Data Chip 风格（secondary_container 青色 / tertiary 危险黄色）
- 状态颜色规范：
  - 未开始：surface 静默
  - 进行中：primary 橙色
  - 已完成：secondary 青色
  - 异常：tertiary 危险黄色（脉冲 0.8→1.0）

------

【七、动效设计（Motion & Feedback）】

- Hover 动效：节点卡片背景偏移至 surface-bright
- 切换动画：节点展开/收起（Mechanical Step-easing）
- 加载动画：CRT 扫描线纹理
- 操作反馈：筛选即时过滤，节点状态变更即时反馈

------

【八、扩展能力（Advanced Features，可选）】

- 场景卡片展示
- 关键节点放大块
- 演练历史记录回溯

------

【九、设计目标（Design Goals）】

- 以时间流为核心，不退化成普通表格页
- 多样化展示演练信息，不能只做简单列表或表格
- 强化过程与阶段感
- 提升演练过程追踪与回溯效率

------

【十、限制与规范（Constraints）】

- 业务来源：原项目"模拟与演练"模块，移除事故响应模块
- 不能虚构新的应急业务闭环
- 必须明确标记：哪些功能是现阶段已有接口能支撑的，哪些内容只是后续可扩展方向
- 不能把后续扩展方向直接写成当前页面功能
- 当前阶段只能基于现有接口能力来做
- 不使用圆角（borderRadius: 0px）
- 不使用纯灰色（#808080）
- 不为了交互完整而凭空增加后端没有的操作流
