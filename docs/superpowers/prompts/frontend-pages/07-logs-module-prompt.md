# 日志模块生成提示词

设计一个【日志模块】，用于【高效率检索系统日志，实现高密度二维表、极强筛选和大数据量承载的专业日志排查工具】。

==================================================

【一、页面定位（Page Role）】

- 页面类型：高密度二维表（专业工具页）
- 用户角色：审计、排查与追踪人员
- 核心目标：
  - 目标1：检索效率（极强筛选，快速定位）
  - 目标2：筛选能力（多条件组合，灵活过滤）
  - 目标3：大量数据可读性（高密度承载，分页/滚动切换）
- 使用场景：审计人员追踪操作记录，排查人员定位异常日志，追踪人员回溯事件链

------

【二、整体布局（Layout System）】

- 顶部结构：固定主导航
- 主体布局：上下分层（筛选工作区 + 日志表格）
- 区域划分：
  - 上方：筛选工作区 + 统计摘要 + 热点日志聚合
  - 中下部：高密度日志表（主角）
- 是否使用特殊布局：Data Grid

------

【三、信息架构（Information Architecture）】

模块1：筛选工作区

- 功能说明：结构清晰的多条件筛选
- 包含信息：多条件组合筛选、快速清空、常用筛选预设
- 优先级：最高（检索入口）

模块2：统计摘要与热点聚合

- 功能说明：顶部统计摘要与热点日志聚合
- 包含信息：日志统计概览、热点日志聚合
- 优先级：中（辅助，不抢表格主视角）

模块3：高密度日志表

- 功能说明：核心日志数据展示
- 包含信息：全量日志记录（高密度二维表）
- 优先级：最高（页面主角）

------

【四、核心模块详细设计（Key Module Design）】

模块名称：高密度日志表

- 展示形式：Data Grid（label-sm 密集矩阵）
- 每个单元包含：
  - 时间戳（Space Grotesk 等宽）
  - 日志级别标识（info / warn / error）
  - 操作模块
  - 操作人
  - 操作描述
  - IP 地址等
- 操作区：
  - 主操作：行点击展开详情
  - 次操作：分页/滚动模式切换
- 状态设计：
  - info：surface 静默
  - warn：tertiary 危险黄色
  - error：error 红

模块名称：筛选工作区

- 展示形式：Recessed 输入区（surface-container-lowest 背景）
- 每个单元包含：
  - 时间范围选择器
  - 日志级别筛选
  - 操作模块筛选
  - 操作人筛选
  - 关键词搜索
- 操作区：
  - 主操作：筛选查询
  - 次操作：快速清空、常用筛选预设
- 状态设计：
  - 筛选激活：secondary 青色标识
  - 筛选为空：surface 静默

------

【五、交互模型（Interaction Model）】

- 主交互方式：筛选 + 表格浏览
- 分层交互：
  - 第一层：筛选工作区设定条件
  - 第二层：表格浏览日志记录
  - 第三层：行点击展开详情
- 页面跳转策略：不跳转，全部在当前页面完成
- 动态行为：筛选即时过滤，分页/滚动模式切换，单页数据量足够大

------

【六、视觉与层级（Visual Hierarchy）】

- 视觉优先级：
  1. 高密度日志表（绝对主角）
  2. 筛选工作区（检索入口）
  3. 统计摘要与热点聚合（辅助）
- 设计风格：Tactical Brutalism — 适度保留传统高密度后台气质，专业工具属性
- 组件风格：
  - 表格：Data Grid（label-sm 密集矩阵，outline-variant 10% 纵向分隔线每4列）
  - 筛选区：Recessed 输入框（surface-container-lowest 背景，focus 时 secondary 青色发光边框）
  - 统计摘要：tonal recession 分区卡片
- 状态颜色规范：
  - info：surface 静默
  - warn：tertiary 危险黄色
  - error：error 红
  - 数值：Space Grotesk 等宽

------

【七、动效设计（Motion & Feedback）】

- Hover 动效：表格行背景偏移至 surface-bright
- 切换动画：分页/滚动切换 Instant（0ms）
- 加载动画：CRT 扫描线纹理
- 操作反馈：筛选即时过滤，无 loading 遮罩

------

【八、扩展能力（Advanced Features，可选）】

- 分页模式与滚动模式切换
- 常用筛选预设保存
- 热点日志聚合（展示层增强）
- 顶部统计摘要

------

【九、设计目标（Design Goals）】

- 不追求花哨排版，重点是检索效率、筛选能力、大量数据可读性
- 即使分页模式，单页数据量也必须足够大
- 不要把日志页改造成信息大屏
- 不要削弱其专业工具属性
- 可以适度保留传统高密度后台气质

------

【十、限制与规范（Constraints）】

- 业务来源：原项目日志记录模块
- 主角始终是表格本身，统计摘要和热点聚合为辅
- 不使用圆角（borderRadius: 0px）
- 不使用纯灰色（#808080）
- 所有功能必须基于现有日志管理接口
- 不为了页面好看而凭空发明后端没有的数据结构
- 展示服从设计系统的冷硬风格，但不削弱专业工具属性
