## 已接入工具

### `resolve_asset_scope`

解析自然语言资产对象。

支持对象：

- 管线名称。
- 管段编号或名称。
- 阀室、站点。
- 设备名称、设备编号、传感器编号。

输出建议：

- 候选对象列表。
- 对象类型。
- 所属区域。
- 置信度。
- 推荐下一步工具。

### `query_area_asset_summary`

区域资产总览。

输出建议：

- 管线数量。
- 管段数量。
- 设备数量。
- 传感器数量。
- 异常设备数量。
- 代表性管线/管段列表。

### `query_asset_relationships`

统一资产关系查询。

作用：

- 查询区域 -> 管线 -> 管段 -> 设备的层级关系。
- 为报告生成“对象上下文”章节。
- 为 ReAct 多工具调用提供后续 object_id。

## 默认策略

当用户说“山东济南的数据”时，不应只返回空对象候选；应返回区域资产摘要，让用户知道济南下有哪些管线、管段和设备可以继续查。
