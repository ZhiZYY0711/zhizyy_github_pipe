# 05-知识库案例与记忆

## 已有能力

| 能力 | 状态 | 作用 | 说明 |
| --- | --- | --- | --- |
| `knowledge_search` | 已接入 ReAct | 检索 SOP、案例、领域知识 | 当前知识库内容刚起步，需要更多运维知识 |
| `memory_search` | 已接入 ReAct | 召回用户偏好和摘要记忆 | 偏交互偏好，业务记忆较少 |
| `search_similar_cases` | 已接入工具 | 查询相似案例 | 可作为快速相似事件入口 |

## 已接入工具或能力

### `query_case_library`

按区域、指标、告警类型、设备类型查询历史案例。

输出建议：

- 案例标题。
- 触发条件。
- 处置动作。
- 结果。
- 与当前问题的相似点。

### `query_sop_by_scenario`

按场景查询 SOP。

场景示例：

- 压力异常。
- 流量突降。
- 传感器离线。
- 阀门状态异常。
- 未闭环告警。

### `build_evidence_pack`

把工具结果和知识库命中合并成证据包。

用途：

- 最终回答引用。
- 报告导出复用。
- 降低导出报告里的内部事件噪声。

### `remember_analysis_preference`

把用户偏好的分析方式沉淀为记忆。

示例：

- “先给区域总览，再给异常明细。”
- “导出报告默认包含证据表。”
- “回答尽量短，结论先行。”

## 默认策略

当业务工具返回数据不足时，Agent 不应马上停止；可以补充 `knowledge_search` 或 `query_sop_by_scenario`，说明“系统数据不足，但按处置规范建议先做什么”。
