## 监测聚合表

### inspection_metric_daily 监测指标日聚合表

按 `stat_date + area_id + pipeline_id + pipe_segment_id` 聚合 `inspection` 明细，用于区域、管道、管段趋势和统计查询。

| 字段名 | 类型 | 限制 | 描述 |
| --- | --- | --- | --- |
| id | bigint | 主键 auto_increment | 聚合记录唯一标识 |
| stat_date | date | not null | 统计日期 |
| area_id | bigint | not null | 区域 ID |
| pipeline_id | bigint | not null | 管道 ID |
| pipe_segment_id | bigint | not null | 管段 ID |
| sample_count | bigint | not null default 0 | 样本数量 |
| pressure_sum | decimal(24, 2) | not null default 0.00 | 压力总和 |
| traffic_sum | decimal(24, 2) | not null default 0.00 | 流量总和 |
| temperature_sum | decimal(24, 2) | not null default 0.00 | 温度总和 |
| shake_sum | decimal(24, 2) | not null default 0.00 | 震动总和 |
| safe_count | bigint | not null default 0 | 安全数量 |
| good_count | bigint | not null default 0 | 良好数量 |
| danger_count | bigint | not null default 0 | 危险数量 |
| critical_count | bigint | not null default 0 | 高危数量 |

索引：`uk_inspection_metric_daily_scope(stat_date, area_id, pipeline_id, pipe_segment_id)`、`idx_inspection_metric_daily_area_date(area_id, stat_date)`、`idx_inspection_metric_daily_pipe_date(pipeline_id, stat_date)`、`idx_inspection_metric_daily_pipe_segment_date(pipeline_id, pipe_segment_id, stat_date)`、`idx_inspection_metric_daily_date_area(stat_date, area_id)`。

## 大屏读模型表

### dashboard_area_metric_daily 大屏区域指标日读模型

按全国、省、市、区四级 `area_id` 汇总 `inspection_metric_daily`，为大屏趋势、告警数量和区域 tooltip 提供快速读取。

| 字段名 | 类型 | 限制 | 描述 |
| --- | --- | --- | --- |
| stat_date | date | 联合主键 | 统计日期 |
| area_id | bigint | 联合主键 | 区域编码，`100000` 表示全国 |
| area_level | varchar(20) | not null | 区域层级：COUNTRY、PROVINCE、CITY、DISTRICT |
| sample_count | bigint | not null default 0 | 样本数量 |
| pressure_sum | decimal(24, 2) | not null default 0.00 | 压力总和 |
| traffic_sum | decimal(24, 2) | not null default 0.00 | 流量总和 |
| temperature_sum | decimal(24, 2) | not null default 0.00 | 温度总和 |
| shake_sum | decimal(24, 2) | not null default 0.00 | 震动总和 |
| safe_count | bigint | not null default 0 | 安全数量 |
| good_count | bigint | not null default 0 | 良好数量 |
| danger_count | bigint | not null default 0 | 危险数量 |
| critical_count | bigint | not null default 0 | 高危数量 |
| refreshed_at | datetime | not null default CURRENT_TIMESTAMP | 刷新时间 |

主键：`(stat_date, area_id)`。索引：`idx_dashboard_area_metric_area_date(area_id, stat_date)`。

### dashboard_area_current_summary 大屏区域当前 KPI 读模型

按全国、省、市、区四级 `area_id` 汇总传感器和任务当前状态，为大屏 KPI 快速读取。

| 字段名 | 类型 | 限制 | 描述 |
| --- | --- | --- | --- |
| area_id | bigint | 主键 | 区域编码，`100000` 表示全国 |
| area_level | varchar(20) | not null | 区域层级：COUNTRY、PROVINCE、CITY、DISTRICT |
| sensor_count | bigint | not null default 0 | 传感器数量 |
| abnormal_sensor_count | bigint | not null default 0 | 异常或离线传感器数量 |
| underway_task_count | bigint | not null default 0 | 进行中任务数量 |
| overtime_task_count | bigint | not null default 0 | 超时或异常任务数量 |
| refreshed_at | datetime | not null default CURRENT_TIMESTAMP | 刷新时间 |

索引：`idx_dashboard_current_level(area_level)`。

### dashboard_alarm_recent 大屏近期告警流水读模型

该表不是聚合表，而是明细级读模型。它把 `inspection` 中 `data_status IN (2, 3)` 的告警记录与 `sensor`、`area` 展示字段提前关联，供大屏报警流水快速读取。

| 字段名 | 类型 | 限制 | 描述 |
| --- | --- | --- | --- |
| inspection_id | bigint | 主键 | 监测记录 ID |
| alarm_time | datetime | not null | 告警时间 |
| sensor_id | bigint | not null | 传感器 ID |
| sensor_name | varchar(100) | not null default '' | 传感器展示名 |
| area_id | bigint | not null | 区域编码 |
| area_name | varchar(255) | not null default '' | 区域展示名 |
| message | varchar(255) | not null default '' | 告警消息 |
| level | varchar(20) | not null default '' | 告警级别 |
| data_status | int | not null | 监测数据状态 |
| pressure | decimal(10, 2) | null | 压力值 |
| traffic | decimal(10, 2) | null | 流量 |
| temperature | decimal(10, 2) | null | 温度 |
| shake | decimal(10, 2) | null | 震动值 |
| refreshed_at | datetime | not null default CURRENT_TIMESTAMP | 刷新时间 |

索引：`idx_dashboard_alarm_area_time(area_id, alarm_time desc)`、`idx_dashboard_alarm_time(alarm_time desc)`。初始化脚本只写入最近 1 天且最多 5000 条；后端 `DashboardReadModelRefreshService` 会在启动后和固定间隔刷新为全量最新 N 条告警，避免基础表短期无更新时快照为空。

### dashboard_read_model_refresh_state 大屏读模型刷新状态表

记录大屏读模型刷新结果。`DashboardReadService.isReadModelAvailable()` 依赖该表判断大屏读模型是否可用。

| 字段名 | 类型 | 限制 | 描述 |
| --- | --- | --- | --- |
| model_name | varchar(80) | 主键 | 读模型名称 |
| source_count | bigint | not null default 0 | 源数据数量 |
| read_model_rows | bigint | not null default 0 | 读模型行数 |
| refreshed_at | datetime | not null default CURRENT_TIMESTAMP | 刷新时间 |
| status | varchar(20) | not null default SUCCESS | 刷新状态 |
| message | varchar(255) | null | 刷新消息 |

索引：`idx_dashboard_refresh_status(status, refreshed_at)`。

## 备注

- 所有外键字段均为逻辑外键，不使用物理外键约束。
- 初始化脚本和性能脚本已经为高频查询建立索引，本文档按当前 DDL 同步列出核心索引。
- `inspection_metric_daily`、`inspection_metric_daily_summary` 属于监测聚合表。
- `dashboard_area_metric_daily`、`dashboard_area_current_summary`、`dashboard_alarm_recent` 属于大屏读模型，其中 `dashboard_alarm_recent` 是明细快照表，不是聚合表。
